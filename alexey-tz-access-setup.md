# ТЗ для Алексея — доступы и настройка под сайт-ассистента «Find Your Floor»

Что это: список технических задач, чтобы разблокировать удалённую сборку сайт-ассистента. Всё можно сделать удалённо, физическое присутствие Олега не нужно. Вопросы по любому пункту — Олегу. Приоритет: блоки 1-3 нужны для старта сборки. Блок 4 (Telegram) нужен позже, под алерты, не срочно.

## Блок 1. WooCommerce — доступ + данные о товарах

### 1.1 REST API ключ (только чтение)

Ассистенту нужен доступ к каталогу на чтение (товары, цены, акции, остатки). 1. WooCommerce → Settings → Advanced → **REST API** → **Add key**. 2. Description: Find Your Floor Assistant. User: твой. **Permissions: Read** (только чтение). 3. Generate → скопировать **Consumer key** и **Consumer secret** (показываются один раз). 4. Передать Олегу защищённо (не в открытом чате, не в гит).

### 1.2 Тег «своего бренда» (Eigenmarke)

Зачем: ассистент должен в первую очередь предлагать **продукцию самого Lux-Floor (своя марка)**, а уже потом перепродаваемые бренды (JANGAL, FALQUON, CHECK, Parador и т.д.). Для этого ему нужно по каждому товару машинно понимать «свой это бренд или нет». 1. Проверь: **есть ли уже** в Woo атрибут или категория «бренд/производитель», по которому можно отличить свою продукцию? Если есть — скажи Олегу, как он называется, ничего больше делать не надо. 2. Если нет — создай **тег товара Eigenmarke** (Products → Tags) и проставь его **всем товарам своей марки Lux-Floor**. Перепродаваемые бренды не трогаем. 3. Если своих SKU много, можно проставить тег массово (bulk edit) по фильтру/категории.

### 1.3 Вес и упаковка (для расчёта доставки)

Ассистент будет сам считать стоимость доставки по весовой таблице с сайта. Для этого по каждому товару нужно знать вес и сколько м² в одной упаковке (VE). 1. Подтверди, где это лежит в Woo: есть ли у товаров заполненное поле **вес (kg)**? 2. И как узнать **м² в одной упаковке (VE)** — отдельное поле, атрибут, или из описания? 3. Просто опиши Олегу, где эти данные, заполнять массово не нужно (если уже есть).

## Блок 2. Bitrix24 — приёмник лидов

Ассистент будет создавать лид в Битриксе по каждому квалифицированному обращению.

### 2.1 Входящий вебхук

1. Битрикс24 → **Разработчикам** → **Другое** → **Входящий вебхук** (Inbound webhook).
2. Права (scope): **crm**.
3. Сохранить → скопировать **URL вебхука** (с токеном). Передать Олегу защищённо.

### 2.2 Поля лида

Создать недостающие пользовательские поля в карточке **Лида** (CRM → Настройки → Поля). Базовые (Имя, Телефон, E-mail) уже есть, нужны дополнительные: - Stadt (город) — если нет - PLZ (индекс) - Quelle / Источник — список, со значением «Website-Assistent» - Interessiertes Produkt (интересующий товар) — текст/строка (SKU) - Menge m² (площадь) — число - Verlegung gewünscht? (нужна укладка) — да/нет - Zubehör-Interesse (аксессуары) — текст, опционально - Geschätzter Auftragswert (оценочная сумма) — число - Lead-Score (HOT/Warm) — список - Profil (профиль из опросника) — текст - DSGVO-Einwilligung (согласие на обработку) — да/нет - Info / заметка — текст > Поля по заказам/упаковке/отгрузке (VPE, Gepackt, Versand, Rechnung) ассистент НЕ трогает — они на стадии сделки.

### 2.3 Ответственные

1. Кто ответственный за новый лид с сайта: **Алиса и Алексей**. Подтвердить, что оба заведены как пользователи и стоят ответственными (тогда Битрикс уведомит сам, push в приложении).
2. Передать Олегу **ID пользователей** Алисы и Алексея (видно в профиле пользователя Битрикса).

## Блок 3. Хостинг (с Олегом)

Решаем тип хостинга (скорее всего managed PaaS, Render/Railway). Если нужен будет доступ к репозиторию/деплою — обсудим отдельно. От тебя на этом шаге ничего, кроме участия в решении.

## Блок 4. Telegram (НЕ срочно, под алерты «горячих» лидов — позже)

Бот и группа Lux Floor **уже существуют** (сделаны при установке CommandOS у Ильи), новый бот создавать не нужно — переиспользуем. Когда дойдём до алертов горячих лидов Илье: 1. Убедиться, что в существующей группе есть тема **🔥 Горячие лиды** (если нет — добавить, бот уже админ). 2. Получить **chat\_id** группы и **thread\_id** этой темы (отправить /id боту в теме). > Этот блок можно отложить, он не блокирует сборку ассистента.

*Передать Олегу: ключи Woo (1.1), ответ по бренду (1.2) и весу/VE (1.3), URL вебхука Битрикса (2.1), ID ответственных (2.3). Этого достаточно, чтобы начать удалённую сборку.*
